## Documented exceptions (no action)

These flagged files are deliberate exceptions per BEST_PRACTICES guidance:

- `components/icons/index.tsx` (438 lines, 49 components) — sanctioned single
  icon module per BEST_PRACTICES §6. Cohesive icon factory pattern, every
  component is a 5-line `<Svg>{paths}</Svg>`. Splitting would scatter the set
  with no readability gain.
- `api/models.ts` (639 lines, 0 components) — pure DTO contract file. No
  logic, no JSX. Hard-limit driven; splitting requires 50+ import sites to
  update and adds no clarity (BEST_PRACTICES §10 puts DTOs in this file).
- `components/ui/Card.tsx` (115 lines, 4 components) — canonical React
  compound-component pattern (`Card.Header = CardHeader`). The "components" are
  members of a single primitive.
- `features/traces/components/TraceTableCells.tsx` (5 components, 58 lines) —
  cohesive cell-renderer module, every cell <20 lines, all share an
  `AgentCallDto` shape. Splitting yields 5 micro-files.
- `features/runs/components/EvaluatorHeatmap.tsx`,
  `features/runs/components/ModelLeaderboardCard.tsx`,
  `features/agents/widgets/ChartsWidget.tsx`,
  `components/search/SearchPreview.tsx` — all inline `style={{}}` blocks are
  genuinely runtime values (per-entity color hashes, runtime grid layout) per
  BEST_PRACTICES §7.
- Scanner false positives (`!` in JSX string literals like `'Copied!'`) —
  `CodeBlock.tsx` and `Invites.tsx` "non-null" counts are spurious.

`TONE_BG_CLS`) so the four refactored proposals files render the same
tokens via Tailwind classes instead of inline style strings.
