# Domain Concepts

The domain (`Proxytrace.Domain/`) currently models:

- **User, Project** — Tenancy. `Project` references one `IModelEndpoint` (`SystemEndpoint`) used by built-in system agents (e.g. agent-name generation, optimizers).
- **Agent** — An AI agent: `Name`, `SystemPrompt` (`IPromptTemplate`), `Tools` (`IReadOnlyList<ToolSpecification>`), `Endpoint` (`IModelEndpoint`), `Project`, `IsSystemAgent` flag.
- **AgentCall** — A captured LLM interaction (one trace entry). Carries an `OutlierFlags` bitmask (`Proxytrace.Domain.AgentCall.OutlierFlags`, stored as a byte) recording which per-call characteristics flagged it: the four statistical outlier bits set at ingestion — high tokens, high latency, low turn-2+ cache hit, many tool calls — plus `CustomAnomaly` (bit 16), set **after** ingestion when a user-defined detector's LLM review returns an anomalous verdict (see `CustomAnomalyDetector` below), and `Blocked` (bit 32), set at ingestion of a call the **proxy rejected in real time** (a blocking detector's trigger matched the request body — the call never reached the provider; attribution lives on the `CustomAnomalyResult`); `None` means none of these. Statistical detection lives in `Proxytrace.Application/Outliers/`: `IOutlierDetector` (per-agent mean ± N·stddev), backed by the bounded-window `IOutlierBaselineReader` and the admin-tunable single-row `IOutlierSettingsStore`. The detector runs inside `AgentCallProcessor` at ingestion; the Traces list filters on `OutlierOnly` or a specific-bit `AnomalyFlags` mask (`AgentCallFilter`). The list also sorts server-side via the `AgentCallSortField` whitelist (created-at, latency, total tokens, tool count, cache-hit rate — the latter two denormalized onto the row at write time). Each call's requested tool names are additionally projected into **AgentCallTool** rows — a storage-only child entity (no domain counterpart, like junction entities) with a denormalized `ProjectId`, backing the traces tool-name filter (`AgentCallFilter.ToolName`, an `EXISTS` semi-join) and the project-scoped distinct tool-name picker; pre-existing traces are backfilled by `AgentCallToolBackfillService` at startup.
- **ModelProvider, Model, ModelEndpoint** — `ModelProvider` is the upstream API (OpenAI, Anthropic, …). `ModelEndpoint` pairs a `Model` with a `ModelProvider` and stores per-token costs (`InputTokenCost`, `OutputTokenCost`); has `CalculateCost(TokenUsage)`.
- **ApiKey** — Proxytrace-issued key for clients hitting the OpenAI proxy. Tied to a `Project` + `ModelProvider`.
- **TestSuite, TestCase** — Curated benchmark inputs. `TestSuite` has N:M with `IEvaluator` (junction `TestSuiteEvaluatorEntity`).
- **TestRun, TestRunGroup, TestResult** — Execution records of a suite against an agent.
- **Evaluator** (base) + concrete subtypes (`IExactMatchEvaluator`, `INumericMatchEvaluator`, `IJsonSchemaMatchEvaluator`, `IToolUsageEvaluator`, `IHelpfulnessEvaluator`, `ISafetyClassifier`, `IPolitenessEvaluator`, `ICustomEvaluator`, plus the LLM-based `IAgenticEvaluator` group). Each `EvaluateAsync(ITestResult)` returns an `IEvaluation` (domain object).
- **OptimizationProposal** — Suggestion to improve an agent: `Kind`, `Status` (Review/Approved/Rejected), `Priority`, `Rationale`, typed `ProposalDetails` (e.g. `SwitchModelProposal`, `UpdateSystemPromptProposal`), `EvidenceTestRunIds`.
- **CustomAnomalyDetector** — A user-defined, LLM-based anomaly detector scoped to a `Project` (`Proxytrace.Domain/CustomAnomaly/`; **Enterprise** — gated by `LicenseFeature.CustomAnomalyDetectors`). `Name`, an `IsEnabled` toggle, 1–20 `Triggers` (`AnomalyTrigger` = `TriggerKind.Phrase` case-insensitive containment, or `TriggerKind.Regex` compiled `IgnoreCase | NonBacktracking` with a 100 ms match timeout), and a scope (`AllAgents`, else a `ScopedAgents` set). The review instructions live as the system prompt of a hidden system `Agent` (`IsSystemAgent`) whose `Endpoint` selects the judge model — mirroring `IAgenticEvaluator`. Pipeline (`Proxytrace.Application/CustomAnomaly/`): after ingestion, non-system calls are queued (`CustomAnomalyReviewService`); a trigger match on the new turn (truncated to 16k chars) gates an LLM review (`skipIngestion`); an anomalous verdict persists an `ICustomAnomalyResult`, sets `OutlierFlags.CustomAnomaly`, publishes the `anomaly-flagged` SSE, and raises an `Anomaly` `Notification` deep-linking the trace. Deleting a detector cascades its results and removes the hidden agent. A detector can additionally set **`BlockUpstream`**: the proxy then evaluates its triggers against the raw request body **before forwarding** and, on a match, rejects the call with an OpenAI-style 403 (`code: proxytrace_blocked`) — the request never reaches the provider. Blocked calls still ingest as traces flagged `OutlierFlags.Blocked` with a `CustomAnomalyResult` attribution; they are not LLM-reviewed (no response to judge). Proxy-side matching is trigger-only (`TriggerMatcher`, now in `Proxytrace.Domain/CustomAnomaly/`), fail-open on infrastructure errors, cached ~30 s, and scoped detectors enforce only when the `x-proxytrace-agent` header names a scoped agent.
- **CustomAnomalyResult** — The record of one anomalous verdict for one call (`Proxytrace.Domain/CustomAnomaly/`): `DetectorId`, `AgentCallId`, `ProjectId`, the `MatchedTrigger` pattern that gated the review, and the judge's `Reasoning` (nullable). Referenced ids are plain `Guid`s; the storage layer cascades results away with their detector or call, so a result never outlives either.
- **Notification** — Multi-purpose, user-facing alert shown on the dashboard (and, in future, other channels): `Kind` (`Anomaly`, `ProposalReady`), `Severity` (Info/Warning/Critical), `Title`, `Message`, `Status` (Unread/Read/Dismissed), optional `ProjectId` scope, and an optional soft `TargetKind`/`TargetId` deep-link reference. Raised today by anomaly detection via `INotificationService`/`INotificationChannel` (`Proxytrace.Application/Notifications/`, `Anomaly/`).
- **Domain objects (no storage):** `IPromptTemplate`, `IPrompt`, `Message` + role-specific subtypes (`SystemMessage`, `UserMessage`, `AssistantMessage`, `ToolMessage`), `Conversation`, `ToolSpecification`/`ToolArguments`/`ToolRequest`/`ToolResponse`, `TokenUsage`, `ICompletion`, `IEvaluation`.
